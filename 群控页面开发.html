# 群控页面开发

  1. 打开新页面 控制页面加载相关参数（地址后面跟的search部分）

group:true|false表示是否是分组控制页 groupId:分组id devTid:设备id ctrlKey:设备控制码 PPK:产品公共秘钥 lang:zh-CN|en-US语言

  1. 页面初始化后 页面初始化后，先按照hekr-h5-sdk文档说明初始化hekr对象，如果是群控sdk会自动切换为群控模式，发送命令还是和单个设备控制一样，如下以控制设备开关为例进行说明

单个设备控制：

$hekr.send({
    cmdTag: 'setSw', // 控制开关的命令
    sw: 1 // 1：打开，0：关闭
})

群控：

$hekr.send({
    cmdTag: 'setSw', // 控制开关的命令
    sw: 1 // 1：打开，0：关闭
})
  1. 下发控制设备命令后，如何获得控制结果？

单个设备控制：

$hekr.on('reportSw', data => {
    console.log(data) // data就是当前控制设备的状态值，由设备上报上来的
})

群控：

$hekr.on('devSend', data => {
    // 理论上在线的设备都会上报一次过来
    // data中包含devTid，我们可以根据devTid区分群组中哪些设备是控制成功的
    // data的数据是设备上报上来的原始数据，其中data传输的设备状态已经解码过了，可直接使用
    console.log(data)
})
  1. 群控群组中的设备管理
    1. 获取群组中的设备：GET https://user-openapi.hekr.me/group
    2. 删除设备：DELETE https://user-openapi.hekr.me/group/{groupId} data=[{ devTid: '', ctrlKey: ''}]
    3. 群组中没有设备了会自动删除群组
